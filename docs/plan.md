# 기획 문서 (Plan)

## 1. 프로젝트 개요

### 목적
대학 과제용 웹사이트로, UI/UX 완성도에 집중하여 프론트엔드만 구현합니다.

### 대상 사용자
- 과제 평가자 (교수님)
- 포트폴리오 열람자

---

## 2. 사이트 맵

```
Main (/)
├── 웹디자인
│   ├── 천 개의 파랑 (/blue-blue)
│   ├── 마루 (/maru)
│   ├── 포트폴리오 (/portfolio)
│   └── 웹사이트 UI (/ui-guide)
└── 기타(준비중)
    └── 기타 페이지 (/etc)
```

---

## 3. 페이지별 요구사항

### 3.1 메인 페이지 (/)
- **목적**: 전체 사이트 네비게이션 허브
- **기능**:
  - 카테고리별 메뉴 표시
  - 각 하위 페이지로의 링크
  - 페이지 전환 애니메이션
- **UX 요소**:
  - 호버 시 카드 확대 효과
  - Stagger 애니메이션으로 순차 등장

### 3.2 천 개의 파랑 (/blue-blue)
- **담당**: 개발자 A
- **테마**: 파랑 계열 색상
- **요구사항**: 추후 정의

### 3.3 마루 (/maru)
- **담당**: 개발자 B
- **테마**: 자연/나무 느낌
- **요구사항**: 추후 정의

### 3.4 포트폴리오 (/portfolio)
- **담당**: 개발자 C
- **테마**: 전문적/미니멀
- **요구사항**: 추후 정의

### 3.5 웹사이트 UI (/ui-guide)
- **담당**: 개발자 D
- **테마**: UI 컴포넌트 쇼케이스
- **요구사항**: 추후 정의

### 3.6 기타 (/etc)
- **담당**: 개발자 E
- **테마**: 자유
- **요구사항**: 추후 정의

---

## 4. 사용자 시나리오

### 시나리오 1: 첫 방문자
1. 메인 페이지 접속
2. 카테고리 확인 (웹디자인, 기타)
3. 관심 있는 페이지 클릭
4. 부드러운 페이지 전환과 함께 해당 페이지로 이동
5. 뒤로가기 버튼으로 메인 복귀

### 시나리오 2: 과제 평가자
1. 메인 페이지에서 전체 구조 파악
2. 각 페이지를 순차적으로 방문
3. UI/UX 완성도 평가
4. 반응형 레이아웃 확인 (데스크탑/모바일)

---

## 5. 인터랙션 명세

### 5.1 페이지 전환
- **방식**: Framer Motion AnimatePresence
- **효과**: Fade in/out with slight vertical movement
- **지속시간**: 300ms

### 5.2 카드 호버
- **효과**: Scale 1.02, Y축 -4px 이동, 그림자 강화
- **지속시간**: 200ms
- **이징**: ease-out

### 5.3 버튼 클릭
- **효과**: Scale 0.98
- **지속시간**: 100ms
